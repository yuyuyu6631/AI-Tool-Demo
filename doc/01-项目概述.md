# 项目概述

## 产品名称

- 中文名：星点评
- 英文名：Xingdianping
- 当前定位：AI 工具发现、筛选、点评、对比与后台运营一体化站点

## 当前定位

当前仓库已经不是“只做前台展示的纯 Demo”。  
它的真实基线是一个正在持续演进的 MVP：

- 公开站点由 `apps/web` 承载
- 真实后端由 `apps/api` 承载
- 页面与数据以真实 API 和数据库模型为准
- 已包含基础认证、用户评论、后台管理入口和榜单运营能力

## 核心价值

平台当前要解决的是“怎么更快帮用户选对 AI 工具”。

围绕这个目标，当前代码已经具备四类核心能力：

1. 目录型发现：按搜索词、分类、标签、价格、访问限制筛工具
2. 决策型详情：进入单工具详情页查看描述、评分、评论摘要、同类替代
3. 场景型推荐：通过场景页、首页推荐区和比较页帮助缩小范围
4. 运营型维护：后台管理工具、评论和榜单，维持内容可更新

## 当前产品形态

### 公开站点

- 首页 `/`
- 工具详情 `/tools/[slug]`
- 场景列表 `/scenarios`
- 场景详情 `/scenarios/[slug]`
- 对比页 `/compare/[comparisonSlug]`
- 认证页 `/auth`
- 匹配页 `/matches`

### 兼容/重定向入口

- `/tools` 会把查询参数透传后永久重定向到 `/`
- `/rankings` 当前直接重定向到 `/`

### 后台入口

- `/admin`
- `/admin/tools`
- `/admin/tools/new`
- `/admin/tools/[id]`
- `/admin/reviews`
- `/admin/rankings`

## 当前用户类型

### 公开站点用户

- 想快速筛选 AI 工具的普通办公用户
- 需要中文可用、价格可控、访问限制明确的轻办公角色
- 需要做工具横向比较的产品、运营、内容和开发角色

### 后台用户

- 负责维护工具目录、榜单和评论内容的管理员

## 当前阶段目标

当前阶段的目标不是“继续证明能不能做”，而是把现有主干稳定下来：

- 让首页成为统一承接入口
- 保持目录、详情、场景、评论和后台口径一致
- 让文档跟着代码自动更新，而不是继续靠手写 PRD 推动
