# 布局改版工作台

> 用途：作为当前前端结构调整的统一口径入口。

## 1. 当前背景

当前站点的主入口已经从“独立目录页承接”收口为“首页统一承接”。  
因此，这轮布局改版的重点不再是把用户导向 `/tools`，而是让首页、详情、场景、对比和后台的视觉与信息层级更一致。

## 2. 本轮目标

1. 稳定首页作为统一承接页的结构
2. 强化详情页、评论区和相似工具的层级
3. 保持后台页与公开站点共享设计语言
4. 让文档口径与当前路由行为一致

## 3. 当前关键事实

- 首页 `/` 是主承接页
- `/tools` 是兼容性重定向
- `/rankings` 是重定向
- 后台页已经存在，不再属于纯规划

## 4. 本轮关注页面

- 首页 `/`
- 工具详情 `/tools/[slug]`
- 场景页 `/scenarios`、`/scenarios/[slug]`
- 对比页 `/compare/[comparisonSlug]`
- 认证页 `/auth`
- 后台页 `/admin/*`

## 5. 本轮不做

- 新推荐算法
- RAG / rerank 深改
- 支付、会员、个人中心
- 与布局无关的数据层大改

## 6. 实施约束

- 首页 URL 参数驱动逻辑不能被破坏
- `/tools` 的兼容重定向行为要继续成立
- 当前评论与后台链路不能因视觉调整退化
- 文档中的未来规划不能冒充当前实现
