# 页面设计规范

> 本文档记录当前实际实现的页面结构，最后更新于 2026-04-07

## 1. 首页 (`/`)

### 页面目标
首页是整站门面，必须让用户一眼明白这是一个帮助做出 AI 工具选择决策的平台，而不是一个"工具链接目录"。核心价值是**降低选错工具的成本**，而不是堆砌工具数量。

### 当前实际包含的模块

#### 1）顶部导航
- Logo
- 首页
- 工具目录
- 严选榜单
- 按场景选
- 用户登录/头像

> 导航右侧持续显示信任标识，格式为：`● 持续更新 · 实测验证可追溯`，使用小号字、低调置于右上角。

#### 2）首屏 Banner
当前实际文案：

**主标题：** 别再无脑搜链接了，告诉我你想干嘛，我帮你挑

**副标题：** 围绕任务目标、使用成本和访问门槛，帮你快速缩小范围，降低选错工具的时间代价

> 语气方向：直接、口语、站在用户角度说话。避免"平台感"、"科技感"等泛化描述，改为"帮你选对、用得上"。

#### 3）首页「AI+搜索」输入框

**Placeholder 文案（轮播或固定）：**
- `免费的 AI 写文案工具，国内能用、不用 VPN…`
- `想做短视频脚本，有没有中文友好的？`
- `帮我找一个学生党能免费用的 AI 编程助手`

**按钮文案：** `帮我挑 →`（不用「搜索」「查询」等中性词）

**输入框下方快速决策词（替换原「快速搜索推荐标签」）：**

标签优先覆盖高频决策维度，而非功能分类：

`职场打工人` `学生党免费` `设计师工具` `副业接单` `中文友好` `新手无脑用` `不用 VPN` `写文案` `做海报` `写代码` `做数据`

> 设计说明：标签优先放"人群 + 访问限制 + 使用目的"，把功能分类标签排在后面。

#### 4）快速决策入口（Banner 区域右侧侧边栏）
展示最多 4 个预设入口，文案要有明确指向性：

- 本周实测热门工具
- 最强 AI 写作严选
- 国内直连 · 无障碍工具合集
- 永久免费可商用工具清单

> 改动说明：原文案偏泛化（"本周热门 AI 工具"），新文案加入**访问限制**和**授权条件**等决策维度，贴近用户真实顾虑。

#### 5）常用分类区

展示最多 4 个热门分类卡片，每个卡片包含分类名称和工具数量。

分类建议（顺序反映用户使用频率）：
- 通用助手
- 写作办公
- 编程开发
- 设计绘图
- 视频音频
- 数据分析
- 智能体平台

#### 6）精选推荐工具卡片区

展示 4 到 8 个工具卡片，每张卡片包含：
- 名称
- 标签（优先展示**人群标签**和**访问限制标签**，其次才是功能标签）
- 评分（视觉权重降低，作为辅助信息，不作为主决策依据）
- 一句话简介（格式建议：`适合 [人群]，可以用来 [具体任务]，[关键限制/优势]`）

> 一句话简介示例：「适合学生党，写作润色效果稳定，免费版每天可用 20 次，国内直连」

#### 7）按人群选

展示 3 到 4 个场景卡片，每个卡片包含场景标题、场景描述、目标人群标签。

文案方向：用**问句**引导，聚焦具体任务而非泛化场景：

- 做 PPT 用什么 AI，这几个够用了
- 写周报、写汇报，打工人用哪个省时间
- 想搭企业知识库，怎么选平台
- 短视频脚本批量生成，哪些工具能商用

> 改动说明：原"按场景推荐"更名"按人群选"，文案改为口语问句，人群标签前置。

#### 8）页脚

包含三个区域：
- 关于星点评：「我们只推真好用的工具，每个推荐背后都有实测记录可查」
- 评测说明：说明评测标准与更新频率
- 协作与反馈：提交工具、报告问题、联系方式

## 2. 榜单与工具目录

### 2.1 严选榜单页 (`/rankings`)

### 页面目标
让用户感受到这是一个有评测逻辑、有筛选标准的内容平台，而不是随便堆砌的工具列表。参考米其林思路：**稀缺、严选、可追溯**。

### 页面内容要求

#### 1）页面标题
**严选榜单**

副标题：`不是热门就上榜，每个工具都经过实测验证`

#### 2）「我们怎么做榜单」说明区

内容方向（简短，不超过 3 条）：
- 每个上榜工具均经过真实场景测试，有测评记录
- 评分是辅助参考，不是上榜唯一依据
- 定期更新，下架或降分的工具会在榜单中标注

#### 3）多个榜单卡片列表

每个工具项包含：
- 排名
- 工具名称
- 一句话简介（格式同首页推荐卡片）
- 标签（人群标签优先）
- 评分（小字展示，辅助信息）
- 推荐理由（重点说明「为什么这个工具上榜」，而不是功能罗列）
- 查看详情链接 + 官网跳转按钮

> 推荐理由示例：「同类工具中中文支持最稳定，免费版限制少，学生和职场人都能直接上手用」

---

### 2.2 工具目录页 (`/tools`)

### 页面内容

#### 1）页面标题 + 介绍

动态展示：工具目录 / 搜索结果

副标题：`按你的真实需求筛，不是按字母顺序排`

#### 2）搜索框

Placeholder：`搜工具名，或者描述你想做的事…`

#### 3）决策快捷筛选（前置展示，重点强化）

原有：最新 / 最热 / 免费优先 / 需付费

**新增前置维度（按决策优先级排序）：**

`最热` `最新` `免费可用` `国内直连` `手机可用` `可商用` `新手友好` `需付费`

> 设计说明：「国内直连」「可商用」「新手友好」是国内用户最高频的决策门槛，应当前置到筛选栏而非藏在详情页标签中。

#### 4）预设榜单快捷入口

保持现有逻辑，文案更新：
- 严选榜单
- 免费工具合集
- 国内可用专区
- 学生党专区

#### 5）左侧筛选区

保留现有字段，调整展示顺序与文案：

**人群标签（新增前置）**
`学生党` `打工人` `设计师` `博主/创作者` `老师` `开发者`

**访问限制**
`国内直连` `需要 VPN` `手机可用`

**价格**
`完全免费` `免费增值` `订阅制` `一次性付费`

**分类**（保持现有多选逻辑）

**排序**：最热 / 最新 / 名称

**标签**（高频标签优先）

**重置筛选**

> 说明：人群标签使用现有 `targetAudience` 字段映射，不改动 API 结构。

#### 6）工具卡片列表

卡片信息层级调整：

1. 工具名称
2. 一句话简介（格式：适合谁 / 能做什么 / 关键限制）
3. 人群标签 + 访问标签（前置展示）
4. 功能标签（次要展示）
5. 评分（最小化展示，辅助信号）

> 弱化「链接感」，卡片点击主体进入详情页，官网链接仅保留小按钮。

#### 7）分页导航

保持现有逻辑。

#### 8）结果统计

显示格式：`找到 XX 个工具，当前第 X 页`

## 3. 工具详情页 (`/tools/[slug]`)

### 页面目标
让用户在详情页能快速判断「这个工具适不适合我」，而不是读完一篇功能介绍还不知道该不该用。

### 当前实际包含的内容

#### 1）基础信息区 ✅ 已实现

包含：工具名称、Logo、一句话定位、综合评分（小字辅助展示）、分类标签、状态标签、返回列表按钮、官网跳转按钮

> 一句话定位文案格式建议：`适合 [人群]，主要用来 [核心任务]，[关键限制或优势]`

#### 2）工具简介 ✅ 已实现

用通俗语言说明这个工具是什么、能做什么，避免官方介绍腔调，优先说「用户会用它来干嘛」。

#### 3）工具信息扩展

包含：开发者、地区、价格模型、支持平台、是否需要 VPN

> **「是否需要 VPN」字段需要视觉强化**，这是国内用户访问前的核心决策门槛，建议用不同颜色标签区分「国内直连」和「需要 VPN」。

#### 4）标签区

展示所有标签，顺序调整为：
1. 适合人群标签（最前）
2. 访问限制标签
3. 推荐场景标签
4. 功能特性标签

#### 5）优点与不足 ❌ 未实现

> 预留位置，v1 不强制实现。后续版本补充「谨慎使用」提示，展示真实局限性。

#### 6）同类工具推荐 ✅ 已实现

展示 2 到 3 个相似工具，标题建议改为：`也有人在用这几个`（比「同类推荐」更口语化）

#### 7）编辑点评 ✅ 已实现

**文案格式建议：**
- 开头说明「适合谁用」
- 中间说「能解决什么具体问题」
- 结尾说「要注意什么」或「和谁相比有什么差别」
- 字数控制在 80 到 120 字，口吻是「用过这个工具的朋友在给你推荐」

## 4. 场景推荐页

### 4.1 场景列表页 (`/scenarios`)

**页面标题：** 按你的真实场景找工具

**页面介绍：** 不按功能分类，按你实际要干的事情来推荐

场景卡片标题用**问句**，例如：
- 做 PPT 用什么 AI 最省事？
- 写公众号文章，哪个工具最顺手？
- 企业想搭知识库问答，怎么选？
- 批量生成短视频脚本，有没有能商用的？

### 4.2 场景详情页 (`/scenarios/[slug]`) ✅ 完全符合

内容结构不变，文案调整方向：

- 场景标题使用问句
- 「解决的核心问题」区块改为「这个场景下你最可能遇到的坑」
- 「优先推荐工具」保持，每个工具的推荐理由聚焦「为什么在这个场景下选它」
- 「备选工具」区块加注：`如果上面的不适合你，可以试试这些`
- 适合人群标签前置

### 页面内容

左右分栏布局，左侧品牌价值展示文案调整：

## 附：信任标识文案规范

全站统一使用以下信任标识文案，不同位置使用不同长度版本：

| 位置 | 文案 |
|------|------|
| 导航右上角（最短） | `● 持续更新 · 实测可追溯` |
| 首页 Badge（短） | `只推真好用 · 不做水测评` |
| 榜单页说明（中） | `不是热门就上榜，每个工具都经过实测验证` |
| 页脚关于区（长） | `我们只推真好用的工具，每个推荐背后都有实测记录可查` |

> 避免使用「权威」「专业」「全面」等空洞词汇，改用可验证的具体承诺。
