# 页面设计规范

> 本文档只记录当前代码已经落地的页面结构与真实路由行为。

## 1. 首页 `/`

### 页面角色

首页已经是当前主承接页，不再只是品牌展示页。

它同时承担：

- 搜索入口
- 筛选结果承接
- 热门/最新/免费/国产等视图切换
- 场景入口
- 热门榜单入口

### 当前模块

1. Header
2. 搜索表单
3. 分类导航 chips
4. 视图切换 tabs
5. 当前筛选 chips 与重置操作
6. 工具列表
7. 分页
8. 场景侧栏
9. 热门榜单侧栏
10. 工具提交入口
11. Footer

### URL 状态

首页当前通过 URL 查询参数承接目录状态，主要包括：

- `q`
- `category`
- `tag`
- `price`
- `access`
- `price_range`
- `sort`
- `view`
- `page`
- `tab`
- `source`

## 2. 兼容路由

### `/tools`

- 当前不是独立承接页
- 保留旧入口
- 会把查询参数透传后永久重定向到 `/`

### `/rankings`

- 当前不是独立榜单页
- 直接重定向到 `/`
- 榜单内容由首页右侧“热门榜单”区域和首页视图承接

## 3. 工具详情页 `/tools/[slug]`

### 当前页面目标

帮助用户快速判断“这个工具适不适合我”。

### 当前已落地内容

1. 返回结果入口
2. 工具基础信息
3. 官网跳转
4. 标签、价格、访问限制、平台支持
5. 编辑点评
6. 评论摘要与评论区
7. 场景推荐
8. 相似工具推荐

## 4. 场景页

### 场景列表 `/scenarios`

当前用于展示场景入口列表。

### 场景详情 `/scenarios/[slug]`

当前用于展示：

1. 场景标题与描述
2. 场景问题说明
3. 主推工具
4. 备选工具
5. 目标人群标签

## 5. 对比页 `/compare/[comparisonSlug]`

### 当前行为

- 支持 2 到 3 个工具横向比较
- 进入页面时会先做 slug 规范化
- 非规范 slug 会重定向到规范地址
- 非法 slug 或工具数不符合要求时返回 `notFound`

## 6. 认证页 `/auth`

### 当前结构

1. 品牌说明区
2. 登录表单
3. 注册表单
4. 社交登录占位按钮
5. 已登录状态卡片

### 当前事实约束

- 当前存在基础账号认证
- 当前仍未接入第三方 OAuth
- 表单校验、提交中状态、错误反馈都已在页面中处理

## 7. 后台页面

当前后台前端已经落地以下路由：

1. `/admin`
2. `/admin/tools`
3. `/admin/tools/new`
4. `/admin/tools/[id]`
5. `/admin/reviews`
6. `/admin/rankings`

### 后台当前职责

- 查看总览数据
- 创建和编辑工具
- 查看和删除评论
- 创建和编辑榜单

## 8. 页面口径约束

- 不再把 `/tools` 写成当前唯一目录承接页
- 不再把 `/rankings` 写成独立榜单页现状
- 不再把“后台管理未落地”写成当前事实
- 不再把“评论未实现”写成当前事实
