# 一期转测试说明：首页统一承接 + 双模式搜索

## 一、当前结论

一期的核心目标已经不再是“把流量导进 `/tools`”，而是：

**把首页 `/` 变成统一承接页，并让搜索、筛选、视图切换、AI 搜索和兼容路由都围绕首页稳定工作。**

## 二、本版测试目标

### 1. 首页是否是唯一主承接页

- 首页承担搜索、筛选、列表和分页
- `/tools` 只保留兼容重定向

### 2. 双模式链路是否可用

- 普通目录检索可用
- AI 搜索链路可用

### 3. 兼容路由是否正常

- `/tools` 是否把查询参数透传后重定向到 `/`
- `/rankings` 是否按当前设计重定向到 `/`

### 4. 现有能力是否未被破坏

- 评论读取/提交
- 详情页跳转
- 对比页
- 场景页

## 三、测试范围

### 已纳入

1. 首页搜索与筛选承接
2. 首页视图 tabs
3. `/tools -> /` 重定向
4. `/api/ai-search`
5. 评论链路
6. 详情页、场景页、对比页跳转

### 不在本轮范围

1. 多轮会话体验
2. OAuth
3. 支付与会员
4. 完整抓取审核发布闭环

## 四、验收标准

### P0

1. 首页主承接可用
2. `/tools` 重定向正常
3. AI 搜索接口失败时页面不崩
4. 评论和详情链路不回归

### P1

1. URL 状态前进/后退一致
2. 视图切换稳定
3. 对比页与场景页跳转稳定
