# UI / UX 设计规范

## 总体视觉方向
整体风格必须符合以下关键词：

- 简洁
- 现代
- 克制
- 稳定
- 平台感
- 轻内容产品风格

**不要**做成传统企业官网，不要过度炫技，也不要出现明显”AI 模板页”气质。

## 推荐参考风格
设计可参考以下方向，但不要直接照搬：

- Linear / Vercel 式克制感
- Notion 式整洁排版
- Stripe / Apple 官网的节奏控制
- 内容产品常见的信息优先级表达

## 页面表现要求
- 首屏要有冲击力
- 首屏不能过满
- 卡片不能太密、太碎、太平均
- 标题层级清楚
- 标签、评分、按钮视觉统一
- 主色控制在 2 到 3 个以内
- 适合快速浏览
- 看上去像”已经有一定基础的平台”

额外要求：

- 基于浅色半透玻璃拟态构建平台感风格
- 通过毛玻璃效果、留白、比例、字重、对齐共同体现审美
- 不依赖过度炫技（不使用大面积渐变、刺眼发光效果）
- 每个区块都要有明确主次，不能同等强调

## 已落地具体设计规范

### 色彩规范

| 类别 | 色值/取值 | 说明 |
|------|-----------|------|
| 页面背景 | `#eef4fb` 淡蓝色 | 配合三色径向渐变点缀（暖橙+浅蓝+淡绿）营造天空感 |
| 主色 | `#11203b` 深蓝近黑 | 唯一主色，用于主按钮和重要标题，文字为白色 |
| 玻璃卡片 | `rgba(255, 255, 255, 0.16~0.5)` | 半透明白色背景，根据区块重要性调整透明度 |
| 边框 | `rgba(255, 255, 255, 0.18~0.28)` | 使用半透明白色边框，而非深色边框 |
| 标题文字 | `slate-950` / `#0f172a` | 高对比度保证可读性 |
| 正文文字 | `slate-600` | 适中对比度减少视觉疲劳 |
| 次要文字 | `slate-500` | 辅助信息、时间、元数据使用 |

**语义标签配色**：使用 Tailwind 语义色板：
- 免费：绿色 `bg-green-100 text-green-800`
- 免费增值：蓝色 `bg-blue-100 text-blue-800`
- 订阅：紫色 `bg-purple-100 text-purple-800`
- 一次性付费：橙色 `bg-orange-100 text-orange-800`
- 已发布：翡翠绿 `bg-emerald-100 text-emerald-800`
- 草稿：琥珀黄 `bg-amber-100 text-amber-800`
- 已归档：灰色 `bg-slate-200 text-slate-700`

### 玻璃拟态核心参数

全站统一使用毛玻璃效果作为核心视觉语言：

```css
backdrop-filter: blur(18px~30px) saturate(140%~150%);
box-shadow: 0 18px 40px rgba(15, 23, 42, 0.08);  /* 柔和弥散阴影 */
border: 1px solid rgba(255, 255, 255, 0.28);     /* 白色半透边框 */
box-shadow: inset 0 1px 0 rgba(255, 255, 255, 0.4);  /* 内发光高光 */
```

可选增强：添加 SVG `turbulence` 滤镜做微小扭曲，增强真实玻璃质感。

### 排版规范

| 层级 | 字号（响应式） | 字重 | 行高 | 使用场景 |
|------|----------------|------|------|----------|
| h1 首页 | `text-4xl md:text-6xl` | 600 (semibold) | `leading-tight` | 首页大标题 |
| h1 内页 | `text-3xl md:text-5xl` | 600 (semibold) | `leading-tight` | 内页页面标题 |
| h2 区块标题 | `text-2xl md:text-3xl` | 600 (semibold) | `leading-tight` | 区块标题 |
| h3 卡片标题 | `text-base` | 600 (semibold) | 正常 | 卡片标题 |
| 正文 | `text-sm / text-base` | 400 (normal) | `leading-6 ~ leading-7` | 正文内容 |
| 标签/小字 | `text-xs / text-[11px]` | 400~500 | 紧凑 | 标签、元数据、辅助文字 |

- 根字号：`16px`
- 字母间距：全大写小字使用 `tracking-[0.24em]` 提高可读性
- 字重原则：标题 500~600，正文 400~500

### 间距系统

所有间距基于 Tailwind 间距系统，保持全站一致：

| 位置 | 间距值（Tailwind） | 像素 |
|------|-------------------|------|
| 区块垂直间距 | `py-12` | 上下各 48px |
| Hero 区域 | `py-16 md:py-20` | 上下 64px / 80px |
| 容器左右内边距 | `px-4 sm:px-6 lg:px-8` | 16px / 24px / 32px |
| 标题到内容间距 | `mt-2 ~ mt-6` | 8px ~ 24px |
| 网格间距 | `gap-4` | 16px |
| 卡片内边距 | `p-5` | 20px |
| 大面板内边距 | `p-6 md:p-8` | 24px ~ 32px |
| Footer 上边距 | `mt-20` | 80px |
| 页面最大宽度 | `max-w-[1440px]` | - |

### 圆角规范

**大圆角是核心特征之一**，根据组件大小使用不同圆角：

| 组件 | 圆角值 |
|------|--------|
| 标签/小按钮 | `rounded-full` |
| 输入框 | `rounded-[18px] ~ rounded-[22px]` |
| 卡片/ToolCard | `rounded-[28px]` |
| 面板/Hero/搜索框 | `rounded-[32px]` |

### 栅格布局系统

响应式栅格遵循以下约定：

| 场景 | 栅格配置 |
|------|----------|
| 4 列卡片网格 | `xl:grid-cols-4 md:grid-cols-2` |
| Hero 主+侧 | `lg:grid-cols-[1.1fr_440px]` |
| Footer | `lg:grid-cols-3` |

### 组件交互规范

- **悬浮效果**：卡片悬浮时 `Y 轴上移 -2px`，阴影增强，不超过 1% 缩放，保持克制
- **过渡动画**：所有过渡 `160ms ~ 180ms` `ease-in-out`，快速响应不拖沓
- **焦点处理**：统一处理 `focus-visible` 外框，保证可访问性
- **动效尊重**：尊重 `prefers-reduced-motion`，关闭非必要动画
- **装饰克制**：仅在 Hero 等核心区域使用径向渐变光斑，不每个卡片都加装饰

### 组件基类约定

项目已沉淀可复用组件基类，新组件应优先使用：

| 类名 | 用途 |
|------|------|
| `page-shell` | 页面外壳，处理背景渐变和最小高度 |
| `site-header` | 网站头部，粘性定位 + 玻璃态 |
| `panel-base` | 通用面板基础样式 |
| `search-panel` | 搜索面板，更强模糊 + 光影 |
| `card-base` | 卡片基础，包含悬浮交互动画 |
| `btn-primary` | 主按钮，深蓝渐变背景 + 白色文字 |

## 配色风格核心原则

使用浅色中性底 + 深色主操作按钮的方案。

重点不是极致设计，而是让页面具备：
- 可信度
- 产品感
- AI 行业调性
