# 愿景落地技术方案（MVO）

> 本文档是未来规划文档，不代表当前代码已经全部具备这些能力。  
> 当前真实实现请优先看 `docs/current-implementation-baseline.md` 与 `doc/08`、`doc/10`、`doc/14`。

## 1. 当前基线已经具备什么

当前代码已经具备的起点能力：

- 公开前端与后台前端都已存在
- 真实后端、真实模型与迁移体系已存在
- 基础认证与管理员权限校验已存在
- 目录、详情、场景、对比、评论、后台管理主链路已存在
- 抓取入口、导入校验和定时任务骨架已存在

这意味着，未来规划不再是“从旧 Demo 迁移到正式主工程”，而是“在现有主干上补齐运营闭环和增长能力”。

## 2. 当前仍缺什么

距离完整 MVO 仍有明显缺口：

- 抓取、审核、发布仍未形成完整闭环
- 后台权限仍较基础，角色体系尚未细分
- OAuth、支付、个人中心未落地
- SEO 专题生产和分发体系未落地
- 行为埋点、归因与效果分析不足
- 共享类型仍未完全统一

## 3. 推荐的未来目标

### 3.1 运营闭环

- 把抓取结果接入审核流
- 把 `tool_updates` 真正用于发布候选管理
- 为后台补齐更多内容状态和审计信息

### 3.2 内容分发

- 建立专题页和 SEO 内容产出链路
- 为榜单、场景和工具详情补齐分发策略

### 3.3 权限与组织能力

- 从 `user / admin` 扩展到更细粒度角色
- 明确编辑、审核、管理员的能力边界

### 3.4 商业化与增长

- 合作链接与线索收集
- 转化归因与数据分析
- 会员或权益体系

## 4. 推荐的演进原则

- 不推翻当前 `Next.js + FastAPI + MySQL` 主干
- 先补运营闭环，再做复杂商业能力
- 未来文档必须持续区分“已实现”和“规划中”
