# 前端落地设计说明

本文档描述当前代码已经落地的前端结构与真实页面行为。

## 1. 当前前端技术基线

- 框架：`Next.js 15 App Router`
- 运行时：`React 19`
- 样式：`Tailwind CSS 4` + `theme.css`
- 测试：`Vitest` + `Playwright`

## 2. 当前页面结构

### 2.1 首页 `/`

首页当前既是品牌入口，也是目录承接页。

已落地区块：

- Header
- 搜索框
- 分类 chips
- 视图 tabs
- 当前筛选 chips
- 工具列表
- 分页
- 场景侧栏
- 热门榜单侧栏
- 工具提交入口
- Footer

### 2.2 兼容目录页 `/tools`

- 当前不是独立目录页
- 会把查询参数透传后重定向到 `/`
- 真实目录承接逻辑已经收口到首页

### 2.3 工具详情 `/tools/[slug]`

已落地能力：

- 基础信息展示
- 官网跳转
- 标签与访问限制展示
- 评论摘要与评论列表
- 编辑点评
- 场景推荐与相似工具

### 2.4 榜单页 `/rankings`

- 当前不是独立内容页
- 直接重定向到 `/`
- 榜单内容由首页右侧区块与首页视图承接

### 2.5 场景页 `/scenarios` 与 `/scenarios/[slug]`

已落地能力：

- 场景列表
- 场景详情说明
- 主推/备选工具组合展示

### 2.6 对比页 `/compare/[comparisonSlug]`

已落地能力：

- 2 到 3 个工具横向比较
- slug 规范化重定向
- 非法组合 `notFound`

### 2.7 认证页 `/auth`

已落地区块：

- 品牌说明区
- 登录/注册表单
- 社交登录占位按钮
- 已登录状态卡片

### 2.8 后台页 `/admin/*`

当前已落地：

- 总览
- 工具管理
- 工具新建/编辑
- 评论管理
- 榜单管理

### 3.2 目录与展示组件

### 3.3 后台组件

- `AdminShell`
- `AdminOverviewDashboard`
- `AdminToolsList`
- `AdminToolEditor`
- `AdminReviewsManager`
- `AdminRankingsManager`
- `AdminAccessGate`

## 4. 当前视觉语言

- 浅色工作台底色
- 白色/半透明卡片层级
- 较大的圆角和低对比边框
- 首页与后台保持统一设计语义，不再拆成两套完全不同的视觉体系

## 5. 当前交互状态

- 搜索与筛选 URL 驱动
- Header 登录态切换
- 移动端菜单开关
- 评论读取与提交
- 后台管理表单与列表交互
- 路由切换反馈

## 6. 当前与历史方案的差异

- 当前不是 `Vite + React Router` 项目
- 当前首页而不是 `/tools` 承接主目录状态
- 当前已存在后台前端和评论前端链路
