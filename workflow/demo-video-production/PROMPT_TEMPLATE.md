# Prompt Template

你要执行“演示视频生产工作流”。

目标：
把当前仓库中的系统演示录制与剪辑操作，按统一流程执行并输出可交付成片。

执行要求：
1. 先确认正确的前端演示入口，而不是默认假设端口可用
2. 本次视频必须采用“菜单总览 + 现有主线”结构
3. 成片前半段必须完整展示桌面端一级导航和工具目录核心菜单
4. 先执行原始录制，产出原始视频、字幕时间轴和包含 scene 元数据的 manifest
5. 再执行剪辑整理，产出演示文案和最终成片
6. 统一通过 `workflow/demo-video-production/run.ps1` 作为调用入口
7. 如果字幕烧录链路不稳定，先保证成片可交付，并保留独立字幕文件

输入信息：
- 仓库根目录：`<在这里填写>`
- 演示入口：`<在这里填写，默认可留空>`
- 是否跳过录制：`<是 / 否>`
- 演示重点：`<在这里填写>`

最终输出要求：
- 说明本次使用的入口地址
- 说明是否重新录制
- 列出原始视频、成片、文案、字幕、manifest 路径
- 说明菜单总览是否完整覆盖
- 说明做过哪些校验
- 说明当前残留限制
